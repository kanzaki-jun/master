NuGet VB2010導入方法

背景

Excelのxlsxファイルの中身を参照したかったので、
COM参照のMicrosoft.Office.Interop.Excelを使ってみたが、
使いづらかった。
具体的には　https://qiita.com/midori44/items/acab9106e6dad9653e73　にも
ある通り、プロセス解放が面倒だった。

そこでClosedXMLを使ってみようとしたところNuGetが必要とわかる。

VB2010の拡張機能マネージャーにはNuGet Packageがあり、
インストールもできるが、そこからNuGetマネージャーを起動できなかった。
原因不明?

そこで、拡張機能マネージャーを使わずにCloseXMLを使用する手順を以下に示す。


導入方法

以下のURLにアクセス
https://www.nuget.org/downloads

このページのWindows x86(64bitならx64) Commandline下のnuget.exeをダウンロード
任意のフォルダを作成し、ダウンロードしたexeをそのフォルダに移動

スタートボタンからコマンドプロンプトを起動

cd 任意のフォルダまでのパス のコマンドを実行して、
任意のフォルダまで移動後

> NuGet.exe install ClosedXML   と入力しコマンドを実行

インストールが終了すると、任意のフォルダ内に
フォルダができるので、その中のClosedXML.0.93.0(.0.93.0は変動する可能性アリ)の中の
libフォルダ内のnet40,net46のどちらかのdllをvb2010の参照で指定

ちなみにnet40,net46は以下の意味を示す

.NET Framework 2.0   --> net20
.NET Framework 3.0   --> net30
.NET Framework 3.5   --> net35
.NET Framework 4.0   --> net40
.NET Framework 4.5   --> net45
.NET Framework 4.5.1 --> net451
.NET Framework 4.5.2 --> net452
.NET Framework 4.6   --> net46
.NET Framework 4.6.1 --> net461
.NET Framework 4.6.2 --> net462
.NET Framework 4.7   --> net47
